# Security Policy

## Supported Versions

Currently, the Red Shield BetaNet is in active development. Security updates are provided exclusively for the latest release.

| Version | Supported          |
| ------- | ------------------ |
| 1.1.0+  | :white_check_mark: |
| < 1.1.0 | :x:                |

## Reporting a Vulnerability

As a post-quantum cryptographic project, we take security vulnerabilities extremely seriously. 

If you discover a security vulnerability within the SIMBA Prime Node, the Quantum Native Wallet, or the underlying ML-DSA implementation, **please DO NOT open a public GitHub issue.**

Instead, please report it via the following channel:
- **Telegram Direct Message:** Contact the core team at [@RedShield_PQC](https://t.me/RedShield_PQC) directly.

### What to include in your report:
- A description of the vulnerability.
- Steps to reproduce the issue (including any relevant logs).
- The version of the Wallet (APK) or Node (MSI) you are using.
- A proposed fix, if you have one.

We will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about our progress. If the vulnerability is accepted, we will coordinate a public disclosure and patch release with you.
